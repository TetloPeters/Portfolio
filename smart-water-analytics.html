Smart Water Analytics

Government Water Usage Analysis & Reporting System for Botswana

Business Intelligence Dashboard Design Government Analytics Data Visualization

Project Overview

The Smart Water Analytics project was developed to provide the Botswana government with comprehensive insights into water usage patterns across various regions. This business intelligence solution enables data-driven decision-making for water resource management, infrastructure planning, and policy development.

Through interactive dashboards and automated reporting systems, government officials can monitor water consumption trends, identify areas of concern, and optimize resource allocation to ensure sustainable water management across the nation.

Data Cleaning Process

The raw water meter data contained over 270,000 records with various quality issues that needed to be addressed before analysis could begin.

Raw Data Challenges

  • Duplicate Records: Found and removed 6,217 duplicate entries while preserving 270,335 unique values
  • Multiple Pulse Types: Data included Pulse1, Pulse1_Total, and Switch2 readings requiring consolidation
  • Time Series Gaps: Irregular timestamps (e.g., 2022-05-02T02:00:01.000Z format) needed standardization
  • Unit Inconsistencies: Mixed measurement units (L for liters, C for temperature) across different series
  • Missing Values: Empty cells and null entries in critical fields required handling
Data Cleaning Approach: Used Excel Power Query and Python pandas to identify duplicates, standardize timestamps, validate unit measurements, handle missing values through interpolation where appropriate, and create consistent field naming conventions across all data sources.

Cleaned Dataset Structure

After cleaning, the dataset included:

  • Timestamp fields in standardized ISO format
  • ManagedObjectId for device tracking (e.g., 21892717, 60193584)
  • Type classifications (Pulse1, Pulse1_Total, DeviceTemperature)
  • Series identifiers (T1, P1) for different measurement types
  • Units (L for water volume, C for temperature)
  • Validated numeric values with outlier detection

Key Performance Indicators

The Power BI dashboard includes four primary visualization panels:

1. Average Value by Month (Pie Chart)
Shows monthly water consumption distribution with 14.6M (23.47%) in May, 16.1M (25.83%) in June, and 13.7M (21.81%) in July, enabling quick identification of peak usage periods.
2. Count of ManagedObjectId by typeM (Line Chart)
Tracks the number of active water meters across different device types over time, revealing patterns in meter deployment and operational status. Shows fluctuations between 0-2K devices with notable variations across different pulse types.
3. Count of Series by Month and Unit (Bar Chart)
Multi-series bar chart displaying measurement counts across May, June, and July for different unit types (Unit, 101-12, din, device, el, sflow, ssule). Helps identify data collection patterns and potential gaps in specific measurement categories.
4. Sum of Value by Time (Area Chart)
Time-series visualization showing cumulative water consumption from April through July 2022, with values reaching up to 27 units. Clear spikes in May, June, and July indicate seasonal usage patterns and help forecast future demand.

Challenges & Solutions

Challenge: Data Quality & Integration

Water usage data came from multiple sources with varying formats and quality standards. This created inconsistencies that could lead to inaccurate analysis.

Solution: Developed robust data cleaning and validation processes using Python scripts. Implemented data quality checks and standardized formats across all data sources to ensure consistency and reliability.

Challenge: User Accessibility

Government officials had varying levels of technical expertise, making it challenging to create dashboards that everyone could use effectively.

Solution: Designed intuitive, user-friendly interfaces with clear visual hierarchies. Provided role-based access with simplified views for non-technical users while maintaining advanced features for analysts.

Challenge: Real-time Updates

Stakeholders needed access to near real-time data for rapid response to emerging issues.

Solution: Implemented automated data refresh schedules and alert systems that notify relevant personnel when critical thresholds are reached or anomalies are detected.
